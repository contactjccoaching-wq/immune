Sonnet benefits most (+58%)

Mid-tier models are the sweet spot: strong enough to follow strategies, weak enough to need them. Sonnet jumped from 17/40 to 28/40 after just 1 immune scan.

Knowledge transfers across domains

Strategies learned from JWT auth code (R1) improved WebSocket, file upload, and proxy code. Security middleware ordering, env validation, and Map-based storage generalize well.

Haiku hits a ceiling

+17% improvement is real but limited. Haiku often fails to implement injected strategies correctly — wrong imports, missing file writes, contradictory validations.

30 antibodies in 8 rounds

The system learned 30 negative patterns and 20 positive strategies from scanning 24 code outputs. Each scan adds ~3 new antibodies and ~2 new strategies on average.

SP+Immune doubles naked baseline

34.6/40 avg — a +104% improvement. The combination is strictly better than either system alone, proving they target different aspects of code quality.

Security is the biggest delta

SP avg 7.3 → SP+IM avg 8.6 (+1.3pp). Immune's antibodies specifically target security gaps (headers, CORS, secrets, rate limiting) that methodology principles alone don't cover.

Immune learning kicks in fast

SP+IM R1 = 29 (same as SP alone), but R2 jumps to 36. After just one scan, the immune system learns enough patterns to push quality near the ceiling.

SP+IM plateaus at 36/40

From R4 onward, SP+IM consistently scores 36/40. The combination reaches a stable near-ceiling with 12 antibodies and 19 strategies learned.

Methodology

Protocol: Reset immune memory to 0. Round 1: generate code without immune (baseline). Rounds 2-8: inject learned cheatsheet strategies before generation, scan output after, learn new patterns.
Models: Haiku (cheapest), Sonnet (mid), Opus (strongest) — each generates the same task independently.
Judge: Opus scores blind on 4 criteria × 10 points = /40 (security, error handling, best practices, robustness).
Tasks: 8 real-world Claude Code tasks across different domains (auth, CRUD, file handling, WebSocket, proxy, CLI, webhooks, session security).
Immune memory: Accumulated 30 antibodies (negative patterns) + 20 strategies (positive patterns) across all rounds.
